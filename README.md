## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ yarn install
```

## Compile and run the project

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Run tests

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ yarn install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

Routes

Public Routes:

    POST /user/auth/register - User registration

    POST /user/auth/login - User login

    POST /admin/auth/login - Admin login

    POST /contact/submit - Contact form submission

User Routes (require user JWT):

    GET /user/profile - Get user profile

    PUT /user/profile - Update user profile

    PUT /user/change-password - Change password

    POST /user/avatar - Upload avatar

    DELETE /user/deactivate - Deactivate account

    POST /user/files/upload - Upload file

    GET /user/files - Get user files

    GET /user/files/:id - Get file by ID

    DELETE /user/files/:id - Delete file

    GET /user/notifications - Get notifications

    GET /user/notifications/unread-count - Get unread count

    PUT /user/notifications/:id/read - Mark as read

    PUT /user/notifications/mark-all-read - Mark all as read

    DELETE /user/notifications/:id - Delete notification

Admin Routes (require admin JWT):

    POST /admin/users - Create user

    GET /admin/users - Get all users

    GET /admin/users/:id - Get user by ID

    PUT /admin/users/:id - Update user

    DELETE /admin/users/:id - Delete user

    PUT /admin/users/:id/status - Change user status

    GET /admin/contacts - Get all contacts

    GET /admin/contacts/stats - Get contact stats

    GET /admin/contacts/:id - Get contact by ID

    PUT /admin/contacts/:id/status - Update contact status

    PUT /admin/contacts/:id/respond - Add response

    GET /admin/contacts/search - Search contacts

    GET /admin/files - Get all files

    GET /admin/files/stats - Get file stats

    GET /admin/files/:id - Get file by ID

    DELETE /admin/files/:id - Delete file

    GET /admin/files/user/:userId - Get user files

    GET /admin/notifications - Get all notifications

    GET /admin/notifications/stats - Get notification stats

    GET /admin/notifications/user/:userId - Get user notifications

    DELETE /admin/notifications/:id - Delete notification
